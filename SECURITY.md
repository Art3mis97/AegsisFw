# Security Policy

## Reporting a Vulnerability

AegisFW is a firewall management tool. Security issues are taken seriously.

If you discover a security vulnerability, please report it responsibly.

### How to Report

**Do not open a public issue for security vulnerabilities.**

Instead, please email: **[TODO: Add security contact email]**

Include:

- A clear description of the vulnerability.
- Steps to reproduce the issue, if possible.
- The potential impact.
- Any suggested fix, if you have one.

### Response

- You will receive an acknowledgement within 72 hours.
- We will work with you to understand and resolve the issue.
- Security fixes will be prioritised over feature work.

### Scope

This policy applies to vulnerabilities in the AegisFW codebase, including:

- The Python management layer.
- Generated nftables rulesets.
- Configuration handling.
- Any privileged operations.

Issues in third-party dependencies should be reported to their respective maintainers.

## Supported Versions

AegisFW is in early development. Security fixes will be applied to the latest development branch only.
